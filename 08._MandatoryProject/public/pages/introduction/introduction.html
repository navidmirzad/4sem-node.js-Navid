What is Node.JS?

  • Lars bak lavede en JS engine der hed V8, som blev opkøbt af google som blev lagt ind i Chrome. Den var enormt effektiv, og virtualiserede ting med memory usage, der gjorde det muligt at skippe en masse steps.

  • V8 var virkelig hurtigt. Historisk set var javascript kun i browseren, men Node.js gjorde det muligt at køre det i backend i stedet for kun frontend som det tidligere havde været.

  • TL;DR - Node.JS is a runtine enviroment that allows us to run JavaScript code on the server-side. And is extremely popuar for building web servers and APIs and many other applications.

Node.JS uses package.json which is a package manager, just like pom.xml in Maven and composer.json in Laravel PHP

Clean Code is a super important principle to keep in mind when building and working on any project within software development.

CC - refers to writing code that is easy to understand, is maintainble an d "easy" to modify. When writing clean code, we wanna avoid unneccesary complexity, to constantly keep the code readable, and make future collaborations easier.

"use strict" enables JavaScript strict mode, which enstricts stricer parsing and error handling on the code runtime.
Basically it helps you write cleaner code, and catch bugs that might have gone unnoticed if you did not have it set on.

ECMAScript - is a scripting language specifically used for scripting / giving web pages functionality.
TL;DR: It is a standardized specification that defines how we write code consistently over different platforms.
To give some perspective, there is also TypeScript which is essentially a stricter ECMAScript.
